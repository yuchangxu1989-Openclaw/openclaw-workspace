# 智能模型路由系统 v1.0
# 设计原则：非关键词匹配，基于任务特征识别

## 路由决策维度

### 1. 任务复杂度评估
```
复杂度指标：
- 代码行数预估
- 架构层级数
- 依赖关系数
- 新颖性（是否标准问题）

低复杂度 (< 3个维度) → Kimi
高复杂度 (≥ 3个维度) → GLM-5
```

### 2. 响应时效要求
```
实时对话 (< 5秒响应) → Kimi
离线任务 (可接受慢响应) → GLM-5
```

### 3. 任务类型特征
```
Kimi擅长：
- 架构设计/规划
- 快速分析
- 对话交互
- 格式整理
- 多轮推理

GLM-5擅长：
- 复杂算法实现
- 代码重构优化
- Bug深度诊断
- 数学/逻辑证明
- 系统性设计
```

### 4. 用户显式指定（最高优先级）
```
用户说"用GLM-5" → 强制使用GLM-5
用户说"用Kimi" → 强制使用Kimi
无指定 → 自动路由
```

## 路由决策流程

```
接收任务
    ↓
[检查用户显式指定] ──是──→ 使用指定模型
    ↓ 否
[评估任务特征]
    - 复杂度评分
    - 时效要求
    - 领域特征
    ↓
[决策]
    ↓
生成执行计划
    ↓
spawn子Agent(选定模型)
```

## 实现方式

不通过关键词匹配，而是通过：
1. **任务分解** - 分析子任务数量和依赖
2. **历史成功率** - 记录哪种模型对该类任务效果更好
3. **资源负载** - 当前哪个模型Key池更空闲

## Key池分配策略

```javascript
// 轮询 + 负载均衡
const selectKey = (model) => {
  const keys = getKeys(model);
  const availableKeys = keys.filter(k => !k.busy);
  
  if (availableKeys.length === 0) {
    // 全部繁忙，选择最近使用的
    return keys.sort((a,b) => a.lastUsed - b.lastUsed)[0];
  }
  
  // 轮询选择
  const index = getNextIndex(model);
  return availableKeys[index % availableKeys.length];
};
```
