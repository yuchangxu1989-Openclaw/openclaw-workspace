## 2. 失败恢复数据流

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          失败恢复数据流                                          │
└─────────────────────────────────────────────────────────────────────────────────┘

  Execution                  LEP Core                Recovery Layer
   Engine                       │                           │
     │                          │                           │
     │ 1. 执行失败              │                           │
     │ {                        │                           │
     │   error: 'timeout',      │                           │
     │   taskType: 'http'       │                           │
     │ }                        │                           │
     │─────────────────────────►│                           │
     │                          │                           │
     │                          │ 2. 记录失败               │
     │                          │ ─[WAL]───────────────────►│
     │                          │                           │
     │                          │ 3. 触发恢复               │
     │                          │ ────────────────────────►│
     │                          │                           │
     │                          │                           │
     │                          │                           │         Auto-Decision
     │                          │                           │           Pipeline
     │                          │                           │               │
     │                          │                           │ 4a. 触发N016 │
     │                          │                           │ ───────────►│
     │                          │                           │ (Pipeline)  │
     │                          │                           │             │
     │                          │                           │             │
     │                          │                           │    CRAS-B   │
     │                          │                           │       │     │
     │                          │                           │ 4b. 触发N017│
     │                          │                           │ ──────►     │
     │                          │                           │ (CRAS)      │
     │                          │                           │             │
     │                          │                           │             │
     │                          │                           │   ISC-DTO   │
     │                          │                           │       │     │
     │                          │                           │ 4c. 触发N018│
     │                          │                           │ ──────►     │
     │                          │                           │ (ISC-DTO)   │
     │                          │                           │             │
     │                          │                           │             │
     │                          │ 5. 恢复结果               │             │
     │                          │ ◄────────────────────────│             │
     │                          │ {                        │             │
     │                          │   recovered: true,       │             │
     │                          │   actions: [...]         │             │
     │                          │ }                        │             │
     │                          │                           │             │
     │                          │                           │             │


  详细恢复流程:

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N016 修复循环                                                            │
  │  ───────────                                                              │
  │  1. Pipeline发现问题 ──► 2. LEP触发N016 ──► 3. 执行修复步骤               │
  │                                              ├── 执行修复                 │
  │                                              ├── 重新扫描                 │
  │                                              └── 评估结果                 │
  │                                                                           │
  │  4. 循环直到: 问题全部修复 或 达到最大迭代次数                            │
  │  5. 返回修复结果                                                          │
  └──────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N017 重复问题根治                                                        │
  │  ─────────────────                                                        │
  │  1. CRAS-B分析 ──► 2. 发现重复模式 ──► 3. LEP触发N017                     │
  │                                                                         │
  │  4. N017执行:                                                             │
  │     ├── 分析重复模式（聚类算法）                                          │
  │     ├── 匹配修复策略                                                      │
  │     │   ├── file_not_found → auto_create_file                             │
  │     │   ├── path_mismatch → auto_update_reference                         │
  │     │   └── skill_not_loaded → trigger_skill_reload                       │
  │     └── 执行修复并标记已解决                                              │
  │                                                                           │
  │  5. 返回解决结果                                                          │
  └──────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N018 全局引用对齐                                                        │
  │  ─────────────────                                                        │
  │  1. 技能重命名/移动 ──► 2. LEP触发N018                                    │
  │                                                                         │
  │  3. N018执行四阶段:                                                       │
  │     Phase 1: 扫描所有目标                                                 │
  │       ├── cron_jobs                                                       │
  │       ├── dto_subscriptions                                               │
  │       ├── isc_rules                                                       │
  │       ├── skill_imports                                                   │
  │       ├── documentation                                                   │
  │       └── capability_anchor                                               │
  │                                                                         │
  │     Phase 2: 创建备份                                                     │
  │       └── 保存到 .backups/n018_<timestamp>/                               │
  │                                                                         │
  │     Phase 3: 批量更新引用                                                 │
  │       └── 按顺序更新所有受影响文件                                        │
  │                                                                         │
  │     Phase 4: 验证完整性                                                   │
  │       ├── no_broken_cron_references                                       │
  │       ├── no_import_errors                                                │
  │       ├── dto_subscriptions_aligned                                       │
  │       └── isc_dto_handshake_pass                                          │
  │                                                                         │
  │     [失败时自动回滚]                                                      │
  │                                                                           │
  │  4. 返回对齐结果                                                          │
  └──────────────────────────────────────────────────────────────────────────┘
```

## 3. 定时调度数据流

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          定时调度数据流                                          │
└─────────────────────────────────────────────────────────────────────────────────┘

  Cron                    LEP                    Rule
Scheduler               Scheduler              Engine
    │                       │                      │
    │                       │                      │
    │ 1. 定时触发           │                      │
    │ (0 */2 * * *)         │                      │
    │──────────────────────►│                      │
    │                       │                      │
    │                       │ 2. 查询规则          │
    │                       │─────────────────────►│
    │                       │  ruleId: 'N017'      │
    │                       │                      │
    │                       │                      │ 3. 条件检查
    │                       │                      │ ────────────
    │                       │ ◄────────────────────│
    │                       │  {                   │ 条件满足?
    │                       │    trigger: true     │
    │                       │  }                   │
    │                       │                      │
    │                       │                      │
    │                       │ 4. 执行规则          │
    │                       │─────────────────────►│
    │                       │                      │
    │                       │                      │ 5. 执行完成
    │                       │ ◄────────────────────│
    │                       │  {                   │
    │                       │    status: 'ok'      │
    │                       │  }                   │
    │                       │                      │
    │ 6. 记录执行           │                      │
    │ ◄─────────────────────│                      │
    │                       │                      │

  调度配置示例:
  ─────────────

  {
    "jobs": [
      {
        "id": "n016-repair-loop",
        "schedule": "*/5 * * * *",     // 每5分钟
        "command": "lep execute-rule N016"
      },
      {
        "id": "n017-recurring-pattern",
        "schedule": "0 */2 * * *",     // 每2小时
        "command": "lep execute-rule N017"
      },
      {
        "id": "n018-alignment-check",
        "schedule": "0 0 * * *",       // 每天
        "command": "lep execute-rule N018"
      }
    ]
  }
```

     │                    │    │ Metrics  │        │
     │  1. 事件流          │    └──────────┘        │
     │ ──────────────────►│         │              │
     │  {                 │         │ 3. 查询      │
     │    type: 'success',│         ▼              │

┌─────────────────────────────────────────────────────────────────────────────────┐
│                        集成系统数据流                                            │
└─────────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────────┐
  │                            parallel-subagent                                 │
  │  ─────────────────                                                           │
  │                                                                              │
  │   Before:                          After:                                    │
  │   ┌──────────────┐                 ┌──────────────┐                         │
  │   │ RetryHandler │                 │  LEPExecutor │                         │
  │   │ CircuitBreak │◄── 改造 ───────►│  (统一入口)  │                         │
  │   │ TimeoutMgr   │                 └──────┬───────┘                         │
  │   └──────────────┘                        │                                 │
  │                                    复用韧性核心                              │
  │                                    (Retry/Circuit/Timeout)                   │
  └─────────────────────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────────────────────┐
  │                       全局自主决策流水线                                     │
