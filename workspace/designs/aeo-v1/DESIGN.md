# AEO智能体效果运营 - 架构设计v1
## Agent Effect Operations Architecture

> **核心定位**: 功能体验准出 + Agent/AI效果准入准出  
> **驱动模式**: 评测与体验驱动  
> **PDCA角色**: C（检查）核心，守门员 + 自主发现问题推动者  
> **关键闭环**: 与自主决策流水线深度集成

---

## 1. 核心使命（双角色）

```
┌─────────────────────────────────────────────────────────┐
│                      AEO 双角色                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────┐         ┌─────────────┐              │
│   │   守门员    │         │  问题发现   │              │
│   │   (Gate)    │◄───────►│  推动者     │              │
│   └──────┬──────┘         └──────┬──────┘              │
│          │                       │                      │
│          ▼                       ▼                      │
│   ┌─────────────┐         ┌─────────────┐              │
│   │  准入准出   │         │  自主决策   │              │
│   │  质量门禁   │         │  流水线触发 │              │
│   └─────────────┘         └─────────────┘              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.1 守门员（Gatekeeper）
- **准入**: 新技能/Agent上线前必须通过AEO评测
- **准出**: 功能发布前效果达标检查
- **阻断**: 不达标自动阻断流水线

### 1.2 问题发现推动者
- **主动发现**: 通过评测数据发现潜在问题
- **自动触发**: 发现问题自动触发N020根因分析
- **闭环解决**: 推动问题进入自主决策流水线修复

## 3. 与多模块联动设计

### 3.1 AEO-CRAS联动（知识沉淀）

```
AEO评测结果 ──→ CRAS知识库
      │
      ├─ 成功案例 → 最佳实践沉淀
      ├─ 失败案例 → 反模式警示
      └─ 效果趋势 → 进化方向输入
```

**联动机制**:
- AEO自动将评测结果推送CRAS
- CRAS基于效果数据优化知识治理
- CRAS学习到的经验反馈AEO评测标准

### 3.2 AEO-SEEF联动（技能进化）

```
AEO发现效果差距 ──→ SEEF技能编排
      │
      ├─ 能力不足 → 触发SEEF创建新子技能
      ├─ 效果回归 → SEEF优化现有子技能
      └─ 场景缺失 → SEEF扩展技能覆盖
```

**联动机制**:
- AEO识别效果短板自动通知SEEF
- SEEF基于差距分析编排新技能
- SEEF子技能上线前必须通过AEO准入

### 3.3 AEO-DTO联动（声明式触发）

```
ISC规则变更 ──→ DTO订阅 ──→ AEO自动更新
      │
      ├─ 评测标准更新
      ├─ 准入门禁调整
      └─ 评测集同步
```

**联动机制**:
- ISC评测标准作为DTO订阅源
- AEO自动接收标准更新
- AEO评测结果通过DTO反馈ISC

### 3.4 AEO-ISC联动（标准双向）

```
ISC标准中心 ◄──────► AEO效果运营
      │                    │
      │ 下发评测标准        │ 反馈效果数据
      │ 更新准入门禁        │ 沉淀评测原则
      │                    │
      ▼                    ▼
   规则更新            标准优化
```

**联动机制**:
- ISC制定效果标准，AEO执行评测
- AEO发现标准不合理，反馈ISC修订
- 形成标准-评测-反馈-优化的闭环

### 3.5 AEO-LEP联动（执行委托）

```
AEO批量评测任务 ──→ LEP执行器
      │
      ├─ 并行执行多个评测用例
      ├─ 韧性保障（熔断/重试/降级）
      └─ 结果收集与聚合
```

**联动机制**:
- AEO生成评测任务批量提交LEP
- LEP保障评测执行的高可用
- AEO接收LEP执行结果生成报告

## 5. 准入准出流程（核心）

### 5.1 技能上线准入流程

```
新技能开发完成
      │
      ▼
┌───────────────┐
│ 开发者自测    │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│ 提交AEO准入申请 │  ★AEO介入
└───────┬───────┘
        │
        ▼
┌───────────────┐
│ AEO准入评测   │  执行完整评测集
│ - 功能正确性  │
│ - 效果达标    │
│ - 性能基准    │
└───────┬───────┘
        │
   ┌────┴────┐
   │ 达标?   │
   └────┬────┘
   是/  \否
   /     \
  ▼       ▼
┌─────┐ ┌───────────────┐
│准出通过│ │ 准出失败      │
│发布上线│ │ 反馈问题清单  │
└─────┘ │ 进入修复流程  │
        └───────────────┘
```

### 5.2 准出标准（Checklist）

| 维度 | 标准 | 检查方式 |
|:---|:---|:---|
| 功能正确性 | 100%核心用例通过 | 自动化评测 |
| 效果达标 | 评分≥阈值 | AEO模型评估 |
| 性能基准 | 延迟<阈值 | LEP监控数据 |
| 韧性可用 | 降级链可用 | 故障注入测试 |
| ISC合规 | 100%规则通过 | N022审计 |
| 文档完整 | SKILL.md达标 | N019检查 |
