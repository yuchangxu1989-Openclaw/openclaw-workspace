## 事件思维模型（2026-03-04 用户亲授）
**事件 = 状态机的触发条件。** 核心教训：
1. 任何对象的生命周期动作（创建/修改/删除）天然就是事件
2. 可量化的条件达到阈值 = 事件（计数器越过阈值的瞬间）
3. 定期扫描+查缺补漏 = 事件体系的兜底网
4. "格式规范"不是不可事件化——执行它的锚点在对象生命周期上
5. 不是不知道事件是什么，是没有认真思考事件监听的锚点/满足条件
6. 事件不只是"发生了什么"，更是"在哪里放探针去捕捉状态变化"
7. 还有组合事件、高度抽象事件等更高级形态——需要继续深入
**这是第一性原理级别的认知，必须贯穿所有架构设计。**
