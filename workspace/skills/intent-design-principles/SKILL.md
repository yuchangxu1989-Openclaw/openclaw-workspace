# 意图理解系统设计原则（正式技能）

## 核心原则
1. **现阶段不过度本地化**：优先构建通用、可迁移的意图体系，避免过早为特定场景做重耦合拆分。
2. **五种收敛类型必须搞定**：
   - 正向情绪
   - 负向情绪
   - 规则意图
   - 复杂意图（5轮上下文推理）
   - 隐含意图（推理）
   - 一句话多意图
3. **AEO 是意图系统的准出门禁**：未达到 AEO（Answer/Execution/Outcome）质量门槛的意图流转不得进入生产闭环。
4. **未知意图发现三段式**：
   - 定期向量聚类
   - LLM 分类
   - MECE 增量识别
   同时对未解决问题进行主动溯源。
5. **意图反熵增红线**：不 MECE 或过于长尾会导致熵增爆炸，必须前置拦截。

## 适用场景
- 新增/下线/重命名意图类型
- 意图系统版本发布前治理审查
- 意图覆盖率巡检
- 未知意图挖掘任务编排
- 意图体系健康度周报/月报

## 操作规范
- 任何意图类型 CRUD 必须经过 MECE 校验。
- 发布前必须满足五种收敛类型全覆盖。
- 新增类型必须通过反熵增检查（MECE + 长尾风险）。
- 未知意图发现任务必须按“聚类→分类→增量识别”生成可调度配置。

## 输出产物
- 意图类型注册表
- 收敛覆盖率检查结果
- 反熵增告警清单
- 未知意图发现任务配置
- 意图类型健康报告（可直接纳入治理评审）
