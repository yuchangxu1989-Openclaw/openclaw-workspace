# 意图理解系统设计原则（用户亲授）

## 核心原则

### 1. 现阶段不过度本地化
优先构建通用、可迁移的意图体系。云端有更好方案，避免过早为特定场景做重耦合拆分；禁止创建"临时""专项""一次性"等过度局部化的意图类型。

### 2. 五种收敛类型必须搞定
意图系统必须对以下五种类型全部支持，缺一不可：

| 类型 | 说明 |
|------|------|
| 正向情绪 (`positive_emotion`) | 用户表达赞扬、满意、感谢等正向情绪 |
| 负向情绪 (`negative_emotion`) | 用户表达抱怨、不满、愤怒等负向情绪 |
| 规则意图 (`rule_intent`) | 可用确定性规则直接匹配的意图 |
| 复杂意图 (`complex_intent_5turn`) | 需要 5 轮上下文推理才能准确识别的意图 |
| 隐含意图 (`implicit_intent`) | 用户未明说但可通过推理得出的意图 |
| 一句话多意图 (`multi_intent_single_utterance`) | 单句话中包含多个独立意图 |

**五种收敛类型是 AEO 准出的前置条件。**

### 3. AEO 是意图系统的准出门禁
AEO（Answer / Execution / Outcome）是意图系统进入生产闭环的质量门禁：
- 未通过 AEO 的意图流转**不得**进入生产
- 通过条件：五种收敛类型全覆盖 + 无熵增风险意图

### 4. 未知意图发现三种方法
按"向量聚类 → 未解决问题溯源 → MECE 增量识别"三段式流程编排：

1. **向量聚类**：对未分类/低置信语料进行密度聚类（HDBSCAN），产出候选意图族群
2. **未解决问题溯源** (`unresolvedBacktrace`)：对历史上未能正确处理的问题进行主动回溯，发现盲区意图
3. **MECE 增量识别**：基于现有注册表互斥穷举校验，识别真正新增意图，须经人工评审

### 5. 意图反熵增红线
**不 MECE 或过于长尾 = 熵增爆炸**，必须前置拦截：
- 新增/更新意图时强制运行 MECE 检查
- 检测关键词重叠、定义相似度过高（Jaccard > 0.78）
- 检测长尾风险：关键词稀少 + 触发率 < 0.3%
- 检测过度本地化：含"临时""专项""一次性""特例"等字样

## 适用场景
- 新增 / 下线 / 重命名意图类型
- 意图系统版本发布前治理审查
- 意图覆盖率巡检
- 未知意图挖掘任务编排
- 意图体系健康度周报 / 月报

## 操作规范
1. 任何意图类型 CRUD 必须经过 MECE 校验
2. 发布前必须满足五种收敛类型全覆盖
3. 新增类型必须通过反熵增检查（MECE + 长尾风险）
4. 未知意图发现任务按"聚类 → 溯源 → 增量识别"生成可调度配置

## index.js 功能（`event.action` 路由）

| action | 说明 |
|--------|------|
| `registry.list` | 列出所有注册意图 |
| `registry.get` | 按 id 查询 |
| `registry.create` | 新增（MECE + 反熵增强制检查） |
| `registry.update` | 更新（同上） |
| `registry.remove` | 删除 |
| `convergence.check` | 五种收敛类型覆盖率检查 |
| `entropy.check` | 反熵增检查（全量或单条） |
| `discovery.schedule` | 生成未知意图发现调度配置 |
| `health.report` | 输出完整健康报告（含 AEO 准出状态） |

## 输出产物
- 意图类型注册表（CRUD 结果）
- 收敛覆盖率检查结果
- 反熵增告警清单
- 未知意图发现三段式调度配置
- 意图类型健康报告（含 AEO 准出门禁状态）
