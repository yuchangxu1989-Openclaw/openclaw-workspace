# LEP 韧性执行中心

## 定位
- **层级**: 基础设施 (infrastructure)
- **用途**: 被CRAS、DTO、ISC、并行子代理等所有业务系统调用
- **核心目标**: 统一分散在各处的韧性执行能力

## 核心功能

### 1. 统一执行接口

```javascript
const { executeRule, execute, health } = require('./skills/lep-executor');

// 执行ISC规则
const result = await executeRule('N016', {
  fixableIssues: [...]
});

// 执行通用任务
const result = await execute({
  type: 'function',
  fn: async () => { ... },
  retryPolicy: { maxRetries: 3 }
});

// 健康检查
const status = await health();
```

### 2. ISC规则执行

**N016 - 修复循环执行器**:
```javascript
const result = await executeRule('N016', {
  fixableIssues: [
    { id: 1, type: 'file_not_found', path: '/path/to/file' }
  ]
});
```

**N017 - 重复问题根治执行器**:
```javascript
const result = await executeRule('N017', {
  // 自动分析最近48小时的事件
});
```

**N018 - 全局引用对齐执行器**:
```javascript
const result = await executeRule('N018', {
  oldName: 'old-skill-name',
  newName: 'new-skill-name'
});
```

### 3. 与现有系统集成

| 系统 | 集成方式 |
|:---|:---|
| parallel-subagent | 复用韧性核心（重试/熔断/连接池） |
| 全局自主决策流水线 | 桥接触发修复流程 |
| ISC-DTO握手 | 桥接对齐检查 |
| CRAS-B洞察 | 事件订阅触发N017执行 |

| `lep_executions_failure` | 失败执行次数 |
| `lep_execution_duration` | 执行耗时 |
| `lep_circuit_breaker_state` | 熔断器状态 |
| `lep_active_executions` | 当前活跃执行数 |

## 注意事项

1. **幂等性**: LEP执行器保证重复执行相同任务不会产生副作用
2. **可观测性**: 所有执行都会记录WAL日志，支持故障排查
3. **回滚机制**: N018等高风险操作支持自动回滚
4. **并发控制**: 内置并发限制，防止资源耗尽
