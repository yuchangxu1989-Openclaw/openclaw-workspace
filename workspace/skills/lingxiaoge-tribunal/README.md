# 凌霄阁-7人裁决神殿 ⚡🏛️

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

> 7位独立Agent，三轮对抗Battle，一个经过充分博弈的裁决。

## 快速开始

```
你：开凌霄阁讨论 [你的议题]
AI：[自动执行三轮协议，输出裁决报告]
```

## 什么是凌霄阁

一个**通用的深度决策技能**。当你面对重大决策、方向分歧、架构取舍时，让7个完全不同视角的"神官"独立分析、互相质疑、最终综合裁决。

**核心价值**：避免单一视角盲区。一个人想不到的漏洞，7个对立视角大概率能找到。

## 7席神官

| 席位 | 视角 | 核心问题 |
|------|------|---------|
| 🏛️ 道席 | 第一性原理 | 根基对不对？ |
| ⚔️ 战席 | 战略取舍 | 该不该做？ |
| 🔧 工席 | 工程实现 | 能不能做？ |
| 🛡️ 盾席 | 风险安全 | 最坏情况？ |
| 👁️ 眼席 | 用户市场 | 有人要吗？ |
| 🔮 远席 | 未来进化 | 3年后呢？ |
| ⚖️ 衡席 | 综合仲裁 | 平衡点在哪？ |

## 三轮协议

1. **Round 1**：7人并行独立审议（不互相看）
2. **Round 2**：看到所有人观点后互相Battle质疑
3. **Round 3**：衡席综合裁决 + 道席审核第一性原理

## 使用方式

详见 [SKILL.md](./SKILL.md)

- **多Agent环境**：7个Agent并行调用
- **单Agent环境**：串行扮演7个角色
- **嵌入流水线**：作为评审终审环节
- **独立调用**：任何需要深度讨论的场景
