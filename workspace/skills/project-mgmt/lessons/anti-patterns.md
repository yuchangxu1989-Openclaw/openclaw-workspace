# 反模式库

每次犯错记录在此，启动项目管理时必须检查。

## 已识别反模式

### AP-001: 凭直觉选工具，不查能力锚点
- **发现日期**: 2026-03-06
- **场景**: 搜索需求默认选web_search(Brave)，忽略已有的tavily-search
- **根因**: 系统prompt近因偏差覆盖实际能力
- **规则**: 任何工具选择前先查CAPABILITY-ANCHOR.md

### AP-002: 硬编码模型名
- **发现日期**: 2026-03-06
- **场景**: ISC规则和cron任务里硬编码claude-opus/sonnet模型名
- **根因**: 创建时图省事直接写死
- **规则**: 模型名跟随openclaw.json配置，写`follow_openclaw_config`

### AP-003: 能力锚点同步脚本硬编码过滤
- **发现日期**: 2026-03-06
- **场景**: 同步脚本排除glm-/cog开头技能，导致27个技能被遗漏
- **根因**: 同步器v1用硬编码黑名单而非全量扫描
- **规则**: 扫描脚本不做排除过滤，全量收录后自动分类
