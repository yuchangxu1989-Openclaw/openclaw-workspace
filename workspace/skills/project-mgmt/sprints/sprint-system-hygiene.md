## Day2: 事件驱动的规则化自动链路
**目标**：五层事件模型（生命周期/阈值/意图/知识发现/系统性模式）任一事件 → 匹配ISC规则 → 自动完成三层代码交付
**交付物**：
1. **事件总线基础设施**：核心事件可被捕获和分发（git hook emit L1事件、CRAS快通道 emit L3意图事件、cron扫描 emit L2阈值事件）
2. **ISC规则匹配引擎**：事件进入 → 匹配规则条件 → 路由到执行层
3. **ruleify执行引擎**：接收任意事件触发后，自动生成三层代码骨架（探针+判断+执行）并注册
4. **多事件源验证**：分别用L1（git commit新技能）、L3（用户自然语言表达规则化意图）验证全链条跑通
**验收标准**：不限于意图事件，任何层级事件都能触发规则化链路；意图识别用LLM语义理解，纳入AEO评测
