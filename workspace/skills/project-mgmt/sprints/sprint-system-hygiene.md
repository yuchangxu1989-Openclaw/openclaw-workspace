## Day2: 意图驱动的规则化自动链路
**目标**：用户任意自然语言表达 → LLM语义理解识别出规则化意图 → 自动完成三层代码交付
**核心认知**：意图识别 = LLM语义理解，不是关键词/正则。"这应该固化"、"以后别犯这错"、"自动化掉" 全是规则化意图。
**交付物**：
1. **CRAS快通道意图提取**：从对话流中实时语义识别意图类型（规则化/查询/执行/反馈等），emit到事件总线
2. **意图→事件映射**：识别出的意图信号（如user.intent.ruleify）触发对应ISC规则
3. **ruleify执行引擎**：接收意图事件后，自动生成三层代码骨架（探针+判断+执行）并注册
4. **端到端验证**：用户自然语言表达 → 意图识别 → 事件触发 → 代码生成 → 注册 → 全链条跑通
**验收标准**：不依赖任何关键词匹配，纯语义理解驱动，对规则化意图的识别准确率纳入AEO评测
