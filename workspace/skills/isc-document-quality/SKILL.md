# ISC文档质量评估

distribution: both


## 功能概述

ISC（智能标准中心）文档质量评估系统，为技能生态提供标准化的文档质量检测与评分服务。

## 评估维度

### 一、基础完整性（40分）

#### SKILL.md 必需字段（20分）
- name：技能名称（5分）
- description：功能描述（5分）
- version: "1.0.1"
- status：状态标识（5分）

#### README.md 基础结构（20分）
- 标题与描述（5分）
- 安装/使用说明（10分）
- 示例代码（5分）

### 二、规范符合度（30分）

#### 命名规范（10分）
- 技能目录：kebab-case（5分）
- 文件命名：符合ISC命名标准（5分）

#### 格式规范（10分）
- 报告输出：文本行格式（禁用Markdown表格）（5分）
- 缩进与排版：统一标准（5分）

#### 语言规范（10分）
- 缩写+中文名：使用"SKILL_SYSTEM.0"、"LEP韧性执行系统"格式（5分）
- 术语一致性：符合SCNM名词缩写表（5分）

### 三、内容准确性（20分）

#### 描述准确性（10分）
- 功能描述与实际代码匹配（5分）
- 使用示例可运行（5分）

#### 时效性（10分）
- 最近更新时间与代码同步（5分）
- 版本号与变更记录匹配（5分）

### 四、扩展完整性（10分）

#### 可选字段（5分）
- created_at：创建时间
- author：作者信息
- dependencies：依赖说明

#### 代码注释（5分）
- 关键函数有docstring
- 复杂逻辑有注释

### 五、设计文档专项评估（20分）

> v1.1.0 新增。专门针对设计文档的深度质量检查，总分20分。

#### 结构合理性（4分）
- 一级目录不超过5章（2分）
- 层级深度不超过3层（2分）

#### 叙事脉络（4分）
- 章节之间有逻辑递进关系（2分）
- 读者能一看就懂，无需反复翻页（2分）

#### 代码分离度（4分）
- 正文代码块密度≤10%（2分）
- 超过5行的代码块已抽到附录（2分）

#### MECE命名一致性（4分）
- 同层级概念命名无重叠（2分）
- 术语全文统一使用（2分）

#### 交叉引用有效性（4分）
- 所有引用目标存在（2分）
- 无悬空引用和循环引用（2分）

**注意：** 启用第五维度后，总分从100分调整为120分，评分标准相应调整。

### 六、重要报告可读性专项评估（20分）

> v1.2.0 新增。针对重要报告、分析报告、决策材料的通用写作钢印检查，总分20分。

#### 中文阅读友好度（4分）
- 是否采用先结论后展开的中文表达（2分）
- 是否避免翻译腔、长句堆叠与术语过载（2分）

#### 思路解释充分度（4分）
- 是否交代判断路径、分析框架、关键假设（2分）
- 是否解释结论背后的取舍依据（2分）

#### 代码降噪度（4分）
- 正文是否避免以代码/命令/接口细节为主体（2分）
- 超过5行的实现片段是否移出正文（2分）

#### 结构与主次（4分）
- 是否具备“结论→背景目标→思路→发现→建议→下一步”的清晰骨架（2分）
- 是否做到重点前置、主次分明（2分）

#### 简洁性（4分）
- 是否删除重复、空话、低信息密度表达（2分）
- 是否做到单段单意、整体不拖沓（2分）

**注意：** 若同时启用设计文档专项评估与重要报告可读性专项评估，总分需在报告头部明确标注为140分制。

## 评分标准（v1.2.0）

适用于包含设计文档专项评估的120分制：

- 108-120分：优秀
- 96-107分：良好
- 84-95分：合格
- <84分：需改进

适用于非设计文档的100分制：

## 评分标准

- 90-100分：优秀
- 80-89分：良好
- 70-79分：合格
- <70分：需改进

## 使用方法

```bash
# 评估单个技能文档
node /root/.openclaw/workspace/skills/isc-document-quality/index.js /path/to/skill

# 评估并生成报告
node /root/.openclaw/workspace/skills/isc-document-quality/index.js /path/to/skill --report
```

## 输出格式

评估报告采用文本行格式，禁用Markdown表格，符合ISC输出规范。
