# 重要报告模板（ISC-REPORT-READABILITY-001）

## 一句话结论
- 先写最关键判断。
- 控制在 1~3 句话内。

## 背景与目标
- 这份报告为什么写。
- 要解决什么问题。
- 不展开无关背景。

## 分析思路
- 说明分析框架。
- 说明关键假设。
- 说明判断依据和取舍逻辑。

## 核心发现
### 发现一（最高优先级）
- 先结论。
- 再给支撑依据。
- 如有数据，只放最关键的数据。

### 发现二
- 先结论。
- 再给支撑依据。

### 发现三
- 先结论。
- 再给支撑依据。

## 建议 / 决策项
- 建议一：写清动作。
- 建议二：写清优先级。
- 建议三：写清预期结果。

## 风险与待确认
- 风险一：影响什么。
- 风险二：还缺什么信息。

## 下一步
- 第一步做什么。
- 第二步做什么。
- 谁负责、什么时候完成（如已知）。

---

### 写作提醒

- 默认中文优先，避免翻译腔。
- 重点讲思路，不堆实现细节。
- 正文少放代码；必要实现细节放附录。
- 一段一件事，避免大段重复。
