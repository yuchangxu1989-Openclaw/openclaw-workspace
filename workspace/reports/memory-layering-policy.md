# 分层记忆与保留边界

## 1. 即时层（Current Turn）
**定义**：仅服务当前请求的临时上下文、草稿、工具中间结果。
**进入条件**：
- 只对当前一步决策有用
- 未被用户声明为偏好、规则、长期事实
- 无跨任务复用价值
**淘汰条件**：
- 当前回复完成即丢弃
- 任务路径改变后立即失效
- 被更正或被新结果覆盖

## 2. 会话层（Session）
**定义**：在本次会话内持续有效的信息，如当前目标、约束、术语映射、已确认方案。
**进入条件**：
- 对后续多轮有直接影响
- 用户已明确确认或持续重复使用
- 属于本次任务范围内的约束、假设、状态
**淘汰条件**：
- 会话结束即清除
- 用户撤回、改口或目标切换
- 长时间未再使用且不影响当前任务

## 3. 持久层（Persistent）
**定义**：跨会话仍应保留的稳定信息，如长期偏好、固定身份信息、长期工作方式。
**进入条件**：
- 用户明确要求“记住”
- 多次稳定重复出现，且跨场景复用
- 低风险、低敏感、长期稳定
**淘汰条件**：
- 用户要求删除或更新
- 信息过时、失真、与新事实冲突
- 超出保留必要性或存在隐私/安全风险

## 4. 禁存层（Do Not Retain）
**定义**：不得进入持久记忆的信息。
**进入条件**：
- 敏感个人数据：证件号、银行卡、密码、验证码、精确住址
- 高风险秘密：密钥、令牌、内部凭据、恢复码
- 短期瞬时数据：一次性链接、临时状态、临时决策噪音
**淘汰条件**：
- 发现即不写入
- 已写入则立即删除
- 出现风险提示时优先清理

## 执行规则
1. **默认最小保留**：能放低层，不升高层。
2. **先用途，后保留**：没有明确复用价值，不进入更高层。
3. **先稳定，后持久**：不稳定、不确定、不重复的信息，不进持久层。
4. **敏感信息默认禁存**：除非法规/业务强制且有明确授权，否则不保留。
5. **可纠正、可删除**：所有非即时记忆都必须支持更新与删除。

## 反例
### 反例 1：把一次性任务细节存成长期偏好
- 错误：用户这次要求“回答简短”，直接记为永久偏好。
- 原因：可能只是当前场景需要，未证明跨会话稳定。

### 反例 2：把敏感凭据写入持久层
- 错误：保存 API Key、验证码、登录 Cookie 以便下次使用。
- 原因：高风险秘密，属于禁存层。

### 反例 3：把未确认推断当作事实保留
- 错误：根据一次聊天推断用户职业或身份并长期保存。
- 原因：未经确认、易失真、可能侵入隐私。

### 反例 4：把中间推理和草稿长期保留
- 错误：将临时候选方案、试错记录、失败命令保存在持久层。
- 原因：仅对当前过程有用，应停留在即时层。

### 反例 5：目标切换后继续沿用旧会话记忆
- 错误：用户改需求后，仍按旧约束执行。
- 原因：会话层信息已失效，应立即淘汰。
