# Open Gaps → 可执行任务清单

> 优先级说明：
> - **P0-可并发立刻发出**：无需等待其他项，可直接派发
> - **P1-短依赖后发出**：依赖轻量对齐后即可执行

## P0-可并发立刻发出

### 1. 记忆分层策略定稿
- **任务名**：定义分层记忆与保留边界
- **推荐角色**：Agent 架构/策略负责人
- **预期产出**：1页策略说明，明确短期上下文/任务记忆/长期记忆的保留范围、淘汰规则、默认不保留项
- **验收标准**：文档包含分层定义、进入条件、淘汰条件、至少3类反例；团队可直接按文实施

### 2. 摘要触发阈值规则落地
- **任务名**：制定长上下文摘要触发器
- **推荐角色**：Prompt/上下文工程师
- **预期产出**：触发规则清单，覆盖 token 阈值、轮次阈值、工具调用阈值、任务切换阈值
- **验收标准**：每个阈值有明确数值与触发动作；可直接接入执行流

### 3. 摘要质量回归校验模板
- **任务名**：设计摘要后回归校验
- **推荐角色**：QA / 评测工程师
- **预期产出**：摘要前后对照检查表，覆盖任务目标、约束、待办、禁做项、关键事实
- **验收标准**：检查表可用于人工或自动校验；至少定义1个“不通过即回滚重摘要”条件

### 4. 常见任务工具预算表
- **任务名**：按任务类型定义工具预算
- **推荐角色**：效率优化 / Agent 运营
- **预期产出**：预算表，覆盖查信息、读文件、执行命令、网页操作等常见任务的调用上限
- **验收标准**：每类任务有默认预算、超预算信号、升级条件；表格可直接贴入规范

### 5. 超预算降级路径
- **任务名**：定义工具超预算后的降级策略
- **推荐角色**：Agent 策略工程师
- **预期产出**：降级剧本，明确“缩小范围 / 停止扩查 / 输出不确定性 / 请求人工介入”的顺序
- **验收标准**：至少覆盖 read/search/exec/browser 四类工具；每类都有停止条件

### 6. 故障恢复树标准化
- **任务名**：产出四类故障恢复剧本
- **推荐角色**：运行稳定性负责人
- **预期产出**：timeout / permission / not-found / flaky 四类恢复树
- **验收标准**：每类包含首选动作、二次尝试条件、最大重试次数、何时停止并上报

### 7. 结束前强制验收清单
- **任务名**：建立任务结束质量门禁
- **推荐角色**：QA / 流程设计
- **预期产出**：文件产出、数据修改、页面操作三类任务的结束前检查清单
- **验收标准**：每类至少3条必检项；未通过不得标记完成

### 8. 复杂任务强制拆分规则
- **任务名**：定义何时必须拆子任务
- **推荐角色**：任务编排/流程设计
- **预期产出**：拆分判定规则，覆盖多工具、多阶段、可回滚、高风险修改场景
- **验收标准**：规则中有明确判定线；给出最小可验证步骤模板

### 9. 核心指标口径统一
- **任务名**：确定5个日常跟踪核心指标
- **推荐角色**：数据分析 / Agent 运营
- **预期产出**：指标定义文档，至少包含 token、工具调用数、首轮成功率、重试率、返工率
- **验收标准**：每个指标有统一定义、计算方式、统计周期、责任人

### 10. 指标面板最小版
- **任务名**：搭建运行指标周更面板
- **推荐角色**：数据工程 / 内部工具工程师
- **预期产出**：可读面板或周报模板，展示核心指标趋势与异常波动
- **验收标准**：可按周更新；能看到环比变化；能挂接优化动作备注

## P1-短依赖后发出

### 11. 摘要机制灰度实验
- **任务名**：对分层记忆+阈值摘要做小流量验证
- **推荐角色**：实验设计 / 评测工程师
- **预期产出**：实验方案与结果记录，比较开启前后 token、响应时延、跑偏率
- **验收标准**：有A/B或前后对照；至少输出1个继续/回滚结论

### 12. 工具无效调用复盘机制
- **任务名**：把无效工具调用纳入周复盘
- **推荐角色**：Agent 运营 / QA
- **预期产出**：复盘模板，记录无效调用类型、原因、可避免规则
- **验收标准**：模板能归因到规则缺失或执行偏差；可直接进入周会

### 13. 恢复剧本接入执行规范
- **任务名**：将恢复树写入执行手册/系统提示
- **推荐角色**：Prompt 工程师 / 规范负责人
- **预期产出**：更新后的执行规范片段
- **验收标准**：timeout / permission / not-found / flaky 的默认动作已写入规范；新任务可直接复用

### 14. 质量门禁嵌入执行流
- **任务名**：把验收清单变成默认结束动作
- **推荐角色**：流程工程师
- **预期产出**：结束动作模板，要求先校验再交付
- **验收标准**：至少覆盖文件、数据、页面三类；执行流中存在明确“未验收不可结束”节点

### 15. 拆分模板接入复杂任务入口
- **任务名**：为复杂请求提供统一拆分模板
- **推荐角色**：任务编排工程师
- **预期产出**：模板，包含目标、约束、步骤、验证点、回滚点
- **验收标准**：复杂任务入口默认可套用；至少给出2个示例

### 16. 指标绑定优化动作
- **任务名**：建立“每项优化必须绑定指标”机制
- **推荐角色**：项目经理 / Agent 运营
- **预期产出**：轻量规则，要求所有优化项标注目标指标与预期变化
- **验收标准**：后续优化条目都能映射到核心指标之一；无法映射的不立项

## 建议派发顺序
1. 先并发发出：1 / 2 / 4 / 6 / 7 / 8 / 9
2. 紧跟并发：3 / 5 / 10
3. 有前置后发：11 / 12 / 13 / 14 / 15 / 16
