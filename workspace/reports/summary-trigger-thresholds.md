# 长上下文摘要触发器阈值设计

适用目标：为执行流提供可直接接入的长上下文摘要触发规则，控制上下文膨胀、减少历史噪声，并在任务切换时保留关键状态。

## 设计原则

1. **先预警、后压缩**：每类阈值分为预警线与强制线。
2. **动作可执行**：每个阈值对应固定动作，避免运行时主观判断。
3. **保留工作记忆**：摘要后必须保留目标、约束、当前进度、未决问题、关键产物路径。
4. **适配工具流**：对高频工具调用单独设阈值，避免工具日志吞噬上下文。

---

## 一、Token 阈值

### T1. Token 预警阈值
- **触发条件**：当前会话累计上下文估算达到 **60,000 tokens**
- **触发动作**：
  1. 立即生成一次 **增量摘要**，仅压缩“已完成步骤、已确认事实、冗长工具输出”。
  2. 保留最近一轮用户要求、当前计划、正在执行的子任务。
  3. 将摘要写入执行流状态中的 `working_summary` 字段。
- **接入建议**：在每次新一轮响应开始前计算一次上下文 token 估算值。

### T2. Token 强制摘要阈值
- **触发条件**：当前会话累计上下文估算达到 **90,000 tokens**
- **触发动作**：
  1. 执行 **强制结构化摘要**。
  2. 丢弃早于最近摘要点的详细推理和大段工具原文，仅保留：
     - 用户目标
     - 约束条件
     - 已完成事项
     - 未完成事项
     - 关键决策
     - 文件/链接/命令产物
     - 风险与待确认项
  3. 用结构化摘要替换旧历史块，继续后续执行。
- **摘要格式要求**：使用固定字段：`goal`、`constraints`、`done`、`next`、`artifacts`、`risks`。

### T3. Token 紧急收缩阈值
- **触发条件**：当前会话累计上下文估算达到 **110,000 tokens** 或达到模型上下文上限的 **85%**（取先到者）
- **触发动作**：
  1. 停止继续展开解释性输出。
  2. 立即执行 **最小保真摘要**。
  3. 只保留：当前目标、最后一次用户消息、最近计划、最近工具结果结论、下一步动作。
  4. 后续所有工具输出仅保留结论，不再回填完整日志。
- **执行优先级**：最高，覆盖其他摘要规则。

---

## 二、轮次阈值

### R1. 轮次预警阈值
- **触发条件**：同一任务连续对话达到 **12 轮**（用户消息 + 助手响应记为 1 轮）
- **触发动作**：
  1. 生成一次 **阶段摘要**。
  2. 标记当前阶段名称，例如：`信息收集`、`实现`、`修复`、`交付`。
  3. 将前 8 轮中已失效的计划项折叠为“历史决策”。

### R2. 轮次强制摘要阈值
- **触发条件**：同一任务连续对话达到 **20 轮**
- **触发动作**：
  1. 强制生成结构化摘要。
  2. 重置短期上下文窗口，仅保留最近 **4 轮原始消息** + 最新摘要。
  3. 若存在待办列表，则重新排序为：进行中 / 阻塞 / 待开始。
- **适用场景**：需求反复迭代、代码修复往返、多轮文案改写。

### R3. 长尾对话收口阈值
- **触发条件**：同一任务连续对话达到 **30 轮**
- **触发动作**：
  1. 自动生成 **任务全局摘要**。
  2. 将历史轮次压缩为里程碑：`起始需求`、`关键变更`、`当前状态`。
  3. 后续执行默认基于全局摘要，不再引用早期原文。

---

## 三、工具调用阈值

### U1. 工具调用频次预警阈值
- **触发条件**：单个任务内累计工具调用达到 **15 次**
- **触发动作**：
  1. 生成一次 **工具结果摘要**。
  2. 对每次工具调用仅保留：`工具名`、`目标`、`结论`、`产物路径/链接`。
  3. 删除冗长 stdout/stderr 原文的历史引用，只保留失败调用的错误摘要。

### U2. 高频工具压缩阈值
- **触发条件**：任意连续 **8 次** 工具调用中，有 **6 次及以上** 属于同一类工具（如 read/exec/browser）
- **触发动作**：
  1. 对该类工具结果执行 **同类折叠摘要**。
  2. 仅保留差异信息，例如：
     - read：仅保留新增发现
     - exec：仅保留退出码、错误、关键输出
     - browser：仅保留页面状态变化和目标元素结果
  3. 在执行流中记录 `tool_burst_summary`。

### U3. 工具调用强制摘要阈值
- **触发条件**：单个任务内累计工具调用达到 **30 次**
- **触发动作**：
  1. 强制生成 **执行证据摘要**。
  2. 压缩全部工具记录为表格化字段：`seq | tool | intent | result | artifact`。
  3. 保留最近 **5 次** 工具调用原始结果，其余仅保留摘要。
- **接入建议**：适合在 agent loop 中用计数器统一控制。

### U4. 大输出单次截断阈值
- **触发条件**：任意一次工具输出超过 **4,000 字符** 或 **800 行**
- **触发动作**：
  1. 不将原始输出全文注入后续上下文。
  2. 即时生成该次调用的 `output_summary`。
  3. 原文仅以外部引用形式保留（如文件路径、日志路径、offset 读取说明）。

## 四、任务切换阈值

### S1. 显式任务切换阈值
- **触发条件**：用户请求中出现新的明确目标，且与当前目标语义相似度低于 **0.45**，或主动作词发生变化（如“修 bug”切换到“写报告”）
- **触发动作**：
  1. 在切换前生成 **前任务收尾摘要**。
  2. 摘要内容至少包括：
     - 原任务目标
     - 完成度
     - 残留待办
     - 关键产物
  3. 将新任务作为新的 `active_task` 启动。
- **实现建议**：可用规则优先，而非完全依赖 embedding。

### S2. 隐式任务切换阈值
- **触发条件**：连续 **3 轮** 对话的核心实体、输出形态、操作对象均发生变化
- **触发动作**：
  1. 自动判定为潜在任务漂移。
  2. 生成一次 **上下文分叉摘要**，分别记录旧任务与新任务。
  3. 后续上下文默认只携带当前分支摘要。

### S3. 多任务并行阈值
- **触发条件**：执行流中同时存在 **2 个及以上** 未完成任务，且最近 **6 轮** 中任务焦点切换达到 **3 次**
- **触发动作**：
  1. 强制生成 **任务面板摘要**。
  2. 采用固定结构：
     - `task_id`
     - `goal`
     - `status`
     - `next_action`
     - `artifact`
  3. 每次响应前仅加载当前焦点任务详情，其余任务只加载一行摘要。

### S4. 任务恢复阈值
- **触发条件**：已切换出去的旧任务在 **10 轮之后** 被重新提起
- **触发动作**：
  1. 不回灌旧原文。
  2. 直接加载该任务最近一次收尾摘要。
  3. 若摘要缺失，则先生成“恢复前摘要”，再恢复执行。

---

## 五、统一触发优先级

当多个阈值同时命中时，按以下优先级执行：

1. **T3 Token 紧急收缩**
2. **S3 多任务并行摘要**
3. **T2 Token 强制摘要**
4. **U3 工具调用强制摘要**
5. **R2 轮次强制摘要**
6. **S1/S2 任务切换摘要**
7. **T1/R1/U1 预警级摘要**

规则：
- 同一轮中最多执行 **1 次强制摘要** + **1 次局部摘要**。
- 若已在最近 **2 轮** 内执行过同类型摘要，则跳过重复触发，除非命中更高等级阈值。

## 六、可直接接入执行流的判定逻辑

```text
if tokens >= min(110000, 0.85 * model_context_limit):
    trigger(T3)
elif multitask_open >= 2 and focus_switches_last_6_turns >= 3:
    trigger(S3)
elif tokens >= 90000:
    trigger(T2)
elif tool_calls_total >= 30:
    trigger(U3)
elif turns_total >= 20:
    trigger(R2)
elif task_switched_explicitly:
    trigger(S1)
elif task_drift_turns >= 3:
    trigger(S2)
elif tokens >= 60000:
    trigger(T1)
elif turns_total >= 12:
    trigger(R1)
elif tool_calls_total >= 15:
    trigger(U1)
```

补充规则：

```text
if tool_output_chars > 4000 or tool_output_lines > 800:
    trigger(U4)

if same_tool_count_last_8 >= 6:
    trigger(U2)

if resumed_old_task_after_10_turns:
    trigger(S4)
```

## 七、推荐摘要模板

### 1. 通用结构化摘要模板
```yaml
goal:
constraints:
done:
current_state:
next:
artifacts:
risks:
```

### 2. 任务面板模板
```yaml
tasks:
  - task_id:
    goal:
    status:
    next_action:
    artifact:
```

### 3. 工具结果摘要模板
```yaml
tool_summary:
  - seq:
    tool:
    intent:
    result:
    artifact:
```

## 八、默认落地建议

建议默认配置如下：

- `token_warn = 60000`
- `token_force = 90000`
- `token_emergency = min(110000, 0.85 * context_limit)`
- `turn_warn = 12`
- `turn_force = 20`
- `turn_global = 30`
- `tool_warn = 15`
- `tool_force = 30`
- `tool_burst = 6/8`
- `tool_output_chars = 4000`
- `tool_output_lines = 800`
- `task_similarity_switch = 0.45`
- `task_drift_turns = 3`
- `multitask_open = 2`
- `focus_switches_last_6_turns = 3`
- `task_resume_gap = 10`

以上阈值可作为默认生产配置，后续可按模型上下文长度和任务类型微调。
