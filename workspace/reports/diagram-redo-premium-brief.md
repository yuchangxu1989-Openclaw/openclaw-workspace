# 图表高质量重绘执行简报

## 1. 上一版失败原因（基于用户反馈）
- **内容缺失**：成图接近空白，没有把原图中的结构、节点、关系和说明真正重建出来。
- **信息密度过低**：只保留了极少元素，无法承担“重绘版”应有的信息传达功能。
- **视觉层级失效**：标题、分区、主次关系、流程方向不清楚，看起来像占位稿而不是可交付成品。
- **可读性不足**：即使有少量元素，也不足以支撑阅读与理解，用户感知为“失败图”。
- **执行流程有问题**：没有先做结构化提取和版式定义，就直接产出图，导致遗漏严重。

## 2. 高质量成品图标准

### 信息密度
- 必须**完整覆盖原图核心信息**：标题、副标题、模块、节点、连接关系、注释、编号/标签。
- 不允许用大面积留白替代内容；留白只服务于分组和阅读节奏。
- 每个视觉区域都应承载明确含义，避免“装饰性空区”。

### 视觉层级
- 至少形成 **4 层层级**：
  1. 页面标题 / 图名
  2. 一级模块 / 分区标题
  3. 关键节点 / 主流程
  4. 辅助说明 / 注释 / 次级关系
- 主流程或主关系必须一眼可见，线条方向和阅读顺序明确。
- 强调重点靠：字号、字重、色块、边框强弱、位置优先级，而不是堆特效。

### 配色
- 采用**低噪声、专业化**配色：
  - 背景：浅色或白底
  - 主色：1 个（用于主流程/关键模块）
  - 辅色：1–2 个（用于分类）
  - 中性色：灰阶用于说明、边框、次级文字
- 避免高饱和杂色、彩虹式配色、过多渐变。
- 颜色必须有语义，不可随意上色。

### 布局
- 优先使用**网格化布局**，保证对齐、间距一致。
- 根据内容类型选择版式：
  - 流程图：单向阅读路径（左→右 / 上→下）
  - 架构图：中心—分支 / 分层结构
  - 对比图：并列分栏
- 模块间距要稳定；连接线避免交叉；必要时通过分区容器降低复杂度。

### 可读性
- 文本必须清晰可读，避免字号过小。
- 文字与背景对比充足，禁止浅灰字压浅色底。
- 节点内文案控制长度，长说明移到注释区。
- 导出图在常见查看比例下，仍能辨识标题、模块名、主连接关系。

## 3. 下一轮重绘的必需工具/流程（避免再次出空白图）

### 必需流程
1. **先做内容拆解**：从原图提取全部文本、分组、层级、连接关系。
2. **先出结构稿**：用文本/草图明确模块数量、相对位置、主阅读路径。
3. **再定视觉规范**：字号、颜色、节点样式、连线样式、间距系统。
4. **最后正式绘制**：严格按结构稿填充，不允许边画边猜。
5. **出图前自检**：逐项对照原图，检查是否有遗漏、空区、断链、弱层级。

### 建议工具
- **首选：矢量/版式工具**（如 Figma、Illustrator、OmniGraffle、ProcessOn、draw.io）
  - 原因：便于精确布局、统一样式、避免生成式图像出现“空白但看似完成”。
- **如需 AI 辅助**：AI 只用于
  - 提取结构
  - 生成文案整理
  - 辅助配色与版式建议
  - **不直接作为最终成图唯一手段**

### 强制检查项
- 成图元素数量是否接近原图信息量
- 是否存在大面积无意义空白
- 是否能一眼看出标题、分区、主流程
- 是否所有关键节点与连接都已落图
- 是否在 100% 查看比例下可读

## 4. 给后续执行者的直接指令
- 不要直接“美化重画”，先完整还原信息结构。
- 不要先追求风格，先保证信息完整与层级正确。
- 不要输出概念海报式大留白画面；目标是**高信息密度、可阅读、可交付的专业图表**。
- 下一步应先产出：**结构清单 + 线框布局**，确认无遗漏后再进入正式重绘。
