# 任务调度效率根治报告

**时间**: 2026-03-11 07:31  
**问题**: 主Agent在Agent池充足时，将多个独立问题打包成一个组合型任务，导致串行执行效率低下

## 问题根因

主Agent调度逻辑缺少**强制并行拆分约束**。现有规则（MEMORY.md "量级→并行"）仅针对量级超标场景，未覆盖"多个独立问题"场景。导致5个无依赖问题被塞进1个子Agent串行处理，浪费Agent池资源。

## 修复内容

### 1. MEMORY.md 铁令区新增规则
- **位置**: `## 🚨 独立任务必须并行派发`（文件末尾，P0最高优先级）
- **核心**: 无依赖关系的独立任务 → 必须拆分为独立子Agent并行执行
- **违反判定**: Agent池充足时打包独立任务 = Badcase

### 2. ISC规则新增
- **文件**: `skills/isc-core/rules/rule.mandatory-parallel-dispatch-001.json`
- **类型**: P0_iron_rule / constraint
- **触发**: task.dispatch.requested（预派发阶段拦截）
- **判断标准**: 子任务间有无输入/输出依赖关系

## 判断逻辑

```
用户请求 → 识别子任务列表
  → 分析依赖关系
    → 无依赖: 强制拆分，每个子Agent一个任务，并行spawn
    → 有依赖: 允许串行/组合，依赖链决定执行顺序
```

## 变更文件清单

| 文件 | 变更 |
|------|------|
| `MEMORY.md` | 新增"独立任务必须并行派发"铁令 |
| `skills/isc-core/rules/rule.mandatory-parallel-dispatch-001.json` | 新增ISC规则 |

## 预期效果

- 5个独立问题 → 5个子Agent并行，耗时从串行5x降至~1x
- Agent池利用率从低效串行提升至满池并行
