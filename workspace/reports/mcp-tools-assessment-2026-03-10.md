# MCP 工具安装评估报告

**日期：** 2026-03-10

## 1. Context7 MCP

| 项目 | 详情 |
|------|------|
| **安装状态** | ✅ 成功 |
| **版本** | @upstash/context7-mcp@2.1.3 |
| **磁盘占用** | 31M |

### 安全评估
- **公网端口暴露：** 否，通过 stdio 通信
- **认证需求：** 无需额外认证，API 调用走 Upstash 公共服务
- **已知漏洞：** 无已知重大漏洞
- **服务冲突：** 无，不监听任何端口，与 Gateway 18789 无冲突
- **隐患：** 会向 Upstash 外部服务发送查询请求，注意数据泄露风险

### 与 OpenClaw 集成可能性
- 可作为 MCP 工具注册，为 Agent 提供**第三方库文档的实时上下文检索**能力
- 适合在编码辅助场景中查询最新 API 文档

## 2. Puppeteer MCP

| 项目 | 详情 |
|------|------|
| **安装状态** | ✅ 成功 |
| **版本** | @modelcontextprotocol/server-puppeteer@2025.5.12 |
| **磁盘占用** | 55M（复用系统已有 Chromium） |
| **端口占用** | 运行时可能使用 9222 (Chrome DevTools Protocol) |
| **进程资源** | 无常驻进程，按需启动；运行时 Chromium 可消耗 200-500MB 内存 |

### 安全评估
- **公网端口暴露：** 默认 localhost，但 CDP 端口(9222)若配置不当可被远程利用
- **认证需求：** CDP 无认证机制，任何能访问该端口的进程均可控制浏览器
- **已知漏洞：** Puppeteer 本身安全，但 headless 浏览器可能被用于恶意用途
- **服务冲突：** 无，与 Gateway 18789 无冲突
- **隐患：** 如果在 `--no-sandbox` 模式运行（root 用户常见），浏览器沙箱被禁用

### 与 OpenClaw 集成可能性
- 可作为 MCP 工具提供**网页抓取、截图、表单填写**等浏览器自动化能力
- 与 OpenClaw 现有 browser 工具功能重叠，可作为补充方案

## 3. Playwright MCP + Chromium

| 项目 | 详情 |
|------|------|
| **安装状态** | ✅ 成功 |
| **版本** | @executeautomation/playwright-mcp-server@1.0.12 + playwright@1.58.2 |
| **磁盘占用** | 189M (MCP server) + 1.8G (Chromium 浏览器) = **~2G** |
| **端口占用** | 无常驻端口 |
| **进程资源** | 无常驻进程；运行时 Chromium 消耗 200-500MB 内存 |

### 安全评估
- **公网端口暴露：** 否
- **认证需求：** 无
- **已知漏洞：** 无已知重大漏洞
- **服务冲突：** 无，与 Gateway 18789 无冲突
- **隐患：** 磁盘占用较大（Chromium 1.8G）；root 运行需 `--no-sandbox`
- **隐患：** 下载的 Chromium 需定期更新以修复浏览器安全漏洞

### 与 OpenClaw 集成可能性
- 比 Puppeteer 更现代，支持多浏览器（Chromium/Firefox/WebKit）
- 可提供**高级网页自动化、测试、数据提取**能力
- 与 OpenClaw 内置 browser 工具互补，适合复杂自动化场景

## 总结

| 工具 | 状态 | 磁盘 | 安全风险 | 推荐优先级 |
|------|------|------|----------|-----------|
| Context7 | ✅ | 31M | 低 | ⭐⭐⭐ 轻量有用 |

### 关键注意事项
1. **无端口冲突** — 所有工具均不与 OpenClaw Gateway (18789) 冲突
2. **无配置修改** — 未触碰 openclaw.json、gateway 或任何现有服务
3. **磁盘消耗** — Playwright Chromium 占 1.8G，是主要开销
4. **后续步骤** — 需在 openclaw.json 中注册 MCP 工具才能使用（由用户决定）
