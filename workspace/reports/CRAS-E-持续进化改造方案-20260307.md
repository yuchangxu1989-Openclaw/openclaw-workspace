# CRAS-E 持续进化化改造方案（2026-03-07）

## 目标
把 CRAS-E 从“每日定时反思任务”升级成“对话驱动 + 事件驱动 + 持久化 + 可验证”的持续进化中枢。

## 这次先打透的桩

### 1. 高优先级意图/进化任务自动落盘
- 新增脚本：`scripts/cras-e-capture.js`
- 能力：
  - 从用户消息中提取高优先级进化任务
  - 自动写入 `memory/tasks/*.json`
  - 自动更新 `PROJECT-TRACKER.md`
  - 自动发出 `system.evolution.opportunity_detected` 事件
  - 自动写入 `memory/cras-e-signals.jsonl`

### 2. 失忆后可恢复的任务状态
- 新增目录：`memory/tasks/`
- 每个关键任务都有独立 JSON 文件
- 启动后不依赖聊天上下文，可直接从任务文件恢复

### 3. CRAS-E 状态报告
- 新增脚本：`scripts/cras-e-status.js`
- 输出：
  - `reports/cras-e/latest-summary.json`
  - `reports/cras-e/latest-summary.md`
- 可直接看当前有多少高优先级桩、优先级分布、未关闭任务数

### 4. CRAS-E 任务注册表
- 新增配置：`skills/cras/config/cras-e-task-registry.json`
- 用于约束哪些任务是必须持续跟踪的“进化硬目标”

## 架构升级路径

### Phase 1：已落地
1. 对话 → 高优先级任务提取
2. 任务 → memory/tasks 持久化
3. 信号 → event bus 发射
4. 状态 → summary 报告

### Phase 2：下一步要落地
1. 每轮对话自动调用 capture，而不是手动脚本触发
2. `memory/tasks` 和 本地任务编排 / dispatcher 自动联动
3. 对 Day2 遗留项建立执行器，不只是记录
4. AEO 评测把“是否真正内化为程序”纳入验收

### Phase 3：终态
1. 对话意图事件成为 CRAS-E 常驻输入流
2. 重复纠偏 / 失忆风险 / 空架子产物自动识别
3. 自动决定：补 rule / 补 test / 补 code / 补 tracker
4. 形成真正的持续进化闭环

## 本次识别出的 P0/P1 任务
- P0 CRAS-E持续进化中枢改造
- P0 失忆后可持续进化保障
- P0 每轮对话意图洞察强制化

## 验收标准
- 不能只写方案，必须有代码 + 状态文件 + tracker 更新
- 不能依赖会话记忆，必须能从 workspace 恢复
- 不能只做日报，必须能形成结构化信号和任务
