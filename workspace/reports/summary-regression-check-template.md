# 摘要后回归校验模板

## 1. 任务目标
- 校验已产出的摘要是否准确反映源材料核心信息。
- 校验摘要是否满足既定范围、格式、风格与输出要求。
- 在发现关键偏差时，触发回滚并重新执行摘要。

## 2. 适用对象
- 任意“先生成摘要、后做回归校验”的任务。
- 包括文档摘要、会议纪要摘要、需求摘要、调研摘要等。

- [ ] 格式符合要求（如列表、分点、段落、表格）
- [ ] 语气符合要求（如客观、中性、正式、简洁）
- [ ] 未泄露不应输出的敏感信息
- [ ] 未引入未经验证的结论或推断

- [ ] 对问题项分类：可直接修订 / 必须重摘要

## 7. 禁做项
- 禁止仅凭印象判定摘要正确，必须对照源材料。
- 禁止为使摘要“更完整”而补写源材料未提供的信息。
- 禁止擅自扩大或缩小摘要范围。
- 禁止忽略用户明确提出的格式、长度、语言限制。
- 禁止将推测、建议、解释伪装成原文事实。
- 禁止在存在关键事实错误时直接放行。

## 8. 判定标准
### 8.1 通过条件
- 关键事实覆盖完整且准确。
- 无重大事实错误、无明显遗漏。
- 满足全部硬性约束。
- 表达清晰，不引发核心语义偏差。

### 8.2 不通过即回滚重摘要条件
满足以下任一项，必须回滚并重新摘要：
- [ ] 出现任一关键事实错误。
- [ ] 遗漏任一用户明确要求必须保留的重点。
- [ ] 摘要范围与任务目标明显不一致。
- [ ] 摘要引入源材料中不存在且影响理解的关键信息。
