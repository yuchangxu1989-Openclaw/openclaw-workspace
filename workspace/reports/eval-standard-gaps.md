# AEO评测标准与基线V3 盲区分析（基于 2026-03-08 实战失败）

## 一、暴露出的关键盲区

1. **缺少“执行证据”维度（说了不做无法被判失败）**
   - 现象：reviewer/analyst 出现“将执行”但无任何工具调用。
   - 盲区：标准偏重计划描述与结果文本，缺少“工具调用证据/事件证据”作为通过前提。

2. **缺少“规则有效性”维度（有规则≠有质量）**
   - 现象：质量门禁规则已落地，但文档仍不达标。
   - 盲区：标准只检查规则是否存在、是否注册，未检查规则是否真实拦截了坏结果。

3. **缺少“新流程采纳率”维度（重构完成但未被使用）**
   - 现象：调度技能重构后，主Agent仍按旧流程执行。
   - 盲区：标准没有“主路径命中率/旧路径调用率”指标，导致惯性回退不被发现。

4. **缺少“任务生命周期闭环完整性”维度（半闭环通过）**
   - 现象：register-task.sh交付，但update-task.sh缺失。
   - 盲区：标准没有把关键后继动作（update/query/close）设为强制检查项。

5. **缺少“产物净化质量”细粒度判定**
   - 现象：飞书文档多轮写入仍有v2残留、重复拼接。
   - 盲区：标准缺少文档结构级质量约束（章节顺序、重复段落、历史残留清除率）。

## 二、判定标准不够细的地方

1. **“完成”定义过宽**
   - 当前：有响应/有汇报即可能被视为完成。
   - 建议：完成=执行证据齐全 + 质量阈值通过 + 复核通过。

2. **“规则落地”定义过宽**
   - 当前：JSON存在、注册完成即可。
   - 建议：落地=声明层+绑定层+执行层+验真层四层全绿。

3. **“质量通过”定义偏主观**
   - 当前：靠review文本判断。
   - 建议：引入可机审指标（重复率、残留率、结构完整率、修复回归通过率）。

## 三、建议新增评测维度（V3→V4）

### 1) 执行真实性（Execution Truthfulness）
- 指标：
  - Tool Invocation Rate（任务期望工具调用覆盖率）
  - Claim-Action Consistency（“将执行”后N秒内是否有工具调用）
- 失败判定：仅文本承诺、无调用证据 = 直接C2。

### 2) 规则有效性（Rule Effectiveness）
- 指标：
  - Rule Trigger Hit Rate（应触发场景命中率）
  - Bad Output Block Rate（坏结果拦截率）
  - False Pass Rate（误放行率）
- 失败判定：规则存在但误放行关键坏结果 = C2。

### 3) 流程采纳与防回退（Adoption & Anti-Regress）
- 指标：
  - New Flow Primary Path Rate（新流程主路径命中率）
  - Legacy Path Leakage（旧流程泄漏率）
- 失败判定：重构后旧路径仍为主路径 = C2。

### 4) 闭环完整性（Lifecycle Completeness）
- 指标：
  - Mandatory Step Coverage（register/update/query/close覆盖率）
  - End-to-End Completion Rate
- 失败判定：缺任一关键后继步骤即降级，不得判通过。

### 5) 文档净化质量（Artifact Hygiene）
- 指标：
  - Legacy Residue Rate（历史版本残留率）
  - Duplication Rate（重复拼接率）
  - Structural Integrity（章节顺序/起始正确率）
- 失败判定：任一超阈值即不通过。

## 四、建议落地到评测集的改造

1. 在 C2 用例模板新增字段：
   - evidence_required（需要的工具/事件证据）
   - quality_gates（必须通过的质量门）
   - anti_regression_checks（防回退检查）

2. 新增四类“强制失败”样例簇：
   - 说了不做（0工具调用）
   - 规则存在但误放行
   - 新流程存在但未采纳
   - 半闭环交付

3. 评测执行器新增 fail-closed 逻辑：
   - 证据缺失即失败
   - 闭环缺步即失败
   - 质量门未过即失败

## 五、优先级建议

- **P0（立即）**：执行真实性 + 闭环完整性 + 文档净化质量
- **P1（本周）**：规则有效性 + 新流程采纳率
- **P2（持续）**：按真实坏例每日自动采集并回灌C2集，维护盲区榜单

## 评测维度补充：角色分离覆盖率

### 定义
每个评测用例是否由独立的质量分析Agent（非执行Agent本身）完成评测。

### 公式
角色分离覆盖率 = 由独立Agent评测的用例数 / 总用例数 × 100%

### 阈值
- 达标：100%（所有用例必须角色分离）
- 不达标：< 100%

### 反作弊
- 同一Agent换label不算分离
- 必须是不同agentId
