## 二、新评测标准更新摘要

### 2.1 闭卷评测硬钢印（ISC-CLOSED-BOOK-001）
核心要求：
- 闭卷评测中**禁止硬编码评测集**。
- **禁止读取记忆、标注、答案、reference、ground truth 等等价参考物**作为判定依据。
- 若存在任一违规，必须：
  - 不得输出“通过 / pass / green / sign-off / 可汇报通过”；
  - 必须输出 **FAIL-CLOSED**；
  - 必须附带失败说明：
    - `closed-book violation: evaluation set hardcoded or reference material accessed`

机器可检合同要求：
- `closed_book_eval.enabled = true`
- `closed_book_eval.no_hardcoded_evalset = true`
- `closed_book_eval.no_reference_reads = true`
- `closed_book_eval.forbidden_paths_checked` 非空
- `closed_book_eval.evidence` 非空
- `gate_status = PASS`

统一失败文案：
- `FAIL-CLOSED: closed-book evaluation violated; hardcoded evalset or reference material access detected.`

### 2.2 LLM 意图识别主基座硬门（ISC-INTENT-EVAL-001）
核心要求：
- 若评测主基座**不是 LLM intent recognition / LLM-based intent understanding**，则不得输出“通过”。
- 必须 **FAIL-CLOSED**。
- 必须明确说明：
  - `缺少 LLM 意图识别主基座`

机器可检证据要求：
- `intent_basis.llm_as_primary = true`
- `intent_basis.evidence` 非空
- `gate` 校验结果为 `PASS`

建议统一失败文案：
- `FAIL-CLOSED: no LLM intent-recognition foundation, cannot report pass.`

## 三、诊断报告整合

### 3.1 通过样例（pass_case）
来源：`sandbox/principle_enforcement_e2e/tests/pass_case.json`

关键结论：
- Final Status: **PASS**
- Release Decision: **ALLOW**
- Gate Recommendation: **通过，允许沙盒准出**

诊断要点：
- `intent_basis.llm_as_primary = true`
- `intent_basis.evidence` 非空
- 自治展开链完整通过：
  - intent / event / isc / dto / cras / aeo / lep / test / gate 全部 PASS
  - release = READY
- 说明该样例满足“LLM 意图识别主基座”要求，且整体门禁校验通过。

### 3.2 失败样例（fail_case）
来源：`sandbox/principle_enforcement_e2e/tests/fail_case.json`

关键结论：
- Final Status: **FAIL**
- Release Decision: **BLOCK**
- Failure Text: `FAIL-CLOSED: no LLM intent-recognition foundation, cannot report pass.`

诊断要点：
- `intent_basis.llm_as_primary = false`
- `intent_basis.evidence = []`
- `intent` 步骤 FAIL
- `gate` 步骤 FAIL
- `release = BLOCKED`

特别发现：
- 失败样例中的 `report_recommendation` 仍为 **“通过”**，与：
  - `final_status = FAIL`
  - `release_decision = BLOCK`
  - `failure_text = FAIL-CLOSED ...`
  明显冲突。
- 这说明**诊断报告产物中存在“推荐语与最终门禁结论不一致”的整改点**。

## 四、整改结论

### 4.1 已明确纳入的新标准
本次评测标准更新后，至少形成了两条硬性门禁：
1. **闭卷评测不得泄漏、不得读取参考物**；否则 FAIL-CLOSED。
2. **没有 LLM 意图识别主基座，不得汇报通过**；否则 FAIL-CLOSED。

### 4.2 诊断产物暴露出的主要问题
当前诊断报告中，失败样例暴露出以下问题：
- **报告推荐语未与最终状态联动**。
- 在 `FAIL` / `BLOCK` 场景下，仍可能出现“通过”字样，存在误导上游汇报风险。

### 4.3 建议整改项
建议后续产物生成逻辑追加以下约束：
- 当 `final_status != PASS` 或 `gate_status != PASS` 时：
  - `report_recommendation` 不得出现“通过 / pass / green / 准出”等正向词。
- 当命中 FAIL-CLOSED 时：
  - 报告正文必须优先展示失败原因；
  - 推荐语应改为“阻断 / 不通过 / 不可准出”。
- 在文档层增加“一致性校验”：
  - `report_recommendation`、`final_status`、`release_decision`、`failure_text` 必须互相一致。

## 五、最终整合结论

### 结论摘要
- **新的评测标准已更新并可归纳为两项硬门禁**：闭卷评测防泄漏、LLM 意图识别主基座要求。
- **诊断报告产物已整合完成**，覆盖通过与失败两类样例。
- **关键整改发现**：失败产物中的推荐语与最终结论不一致，需要修复报告生成逻辑。

### 对外可读表述
当前评测体系整改后，已经把“闭卷评测防作弊”和“LLM 意图识别主基座”上升为不可豁免的硬门。诊断结果显示，通过样例满足门禁要求；失败样例能够被正确阻断，但失败报告仍存在推荐语错误，说明报告生成链路还需要补齐一致性约束，避免把阻断结果误写成“通过”。
